# AGENTS.md

본 문서는 BAIK 버스정류장 키오스크 서비스의 프론트엔드 개발을 위해  
AI(Codex)가 반드시 따라야 할 **최상위 행동 규칙**을 정의한다.

Codex는 이 문서를 기준으로 판단하고,  
추측 없이 명시된 문서와 규칙만을 사용하여 코드를 생성한다.

---

## 1. 프로젝트 개요

BAIK는 버스 정류장에 설치되는 키오스크 기반 대중교통 안내 서비스이다.

- 짧은 사용 시간
- 큰 화면
- 터치 및 음성 기반 인터페이스
- 빠르고 직관적인 정보 제공

이 프로젝트는 **Vue 3 + TypeScript + Pinia** 기반의 프론트엔드로 구현된다.

---

## 2. Codex의 역할

Codex는 이 프로젝트에서 다음 역할을 수행한다.

- 화면 명세에 맞는 Vue 컴포넌트 생성
- project_convention.md에 정의된 규칙을 준수한 코드 작성
- API 모델 구조를 정확히 반영한 데이터 처리 코드 작성

Codex는 **기획자나 설계자가 아니며**,  
정의되지 않은 구조나 동작을 임의로 생성하지 않는다.

---

## 4. 프론트엔드 구현 규칙 요약

- Vue 3 `<script setup>` 방식만 사용한다
- `<style scoped>`를 기본으로 한다
- 전역 상태는 반드시 Pinia Store로 관리한다
- 컴포넌트 내부에서 전역 상태를 직접 생성하지 않는다
- 컴포넌트는 최대한 재사용 가능하게 작성한다
- 팝업은 공통 컴포넌트로 구성한다
- 반응형 전역 변수는 `g_` 접두사를 사용한다
- 변수명, 함수명은 **의미가 명확해야 한다**

---

## 5. API 및 데이터 참조 규칙

- API 요청/응답 구조는 `_variableSpec/api-models`의 Python 모델을 기준으로 한다
- API 응답 데이터를 임의로 가공하지 않는다
- `payload`, `data`, `response`와 같은 추상적인 변수명 사용을 금지한다
- `bus_key`, `bus_key_list`와 같이 **의도가 드러나는 명칭**을 사용한다

---

## 6. 절대 금지 사항

Codex는 다음 행동을 하지 않는다.

- 화면 명세에 없는 UI 요소 생성
- API 모델에 없는 필드 추측
- 규칙 문서에 없는 코드 스타일 생성
- 임의의 상태 관리 방식 도입
- 옵션 API 사용
- 규칙을 무시한 컴포넌트 분리

---
