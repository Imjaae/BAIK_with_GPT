# BAIK 서비스 설명

BAIK는 **버스 정류장에 설치되는 키오스크 기반 대중교통 안내 서비스**이다.  
버스를 기다리는 시민이 **복잡한 조작 없이, 짧은 시간 안에 필요한 이동 정보를 확인**할 수 있도록 설계되었다.

본 서비스는 터치 조작과 음성 안내를 중심으로 동작하며,  
정류장이라는 한정된 공간과 사용 환경을 고려하여 **직관성, 가독성, 즉시성**을 최우선 가치로 한다.

---

## 서비스 목적

- 버스 정류장에서 대기 중인 사용자가  
  **도착 버스 정보와 이동 경로를 빠르게 확인**할 수 있도록 지원
- 스마트폰 사용이 익숙하지 않은 사용자도  
  **키오스크 조작만으로 목적지 탐색 가능**
- 실시간 교통 정보와 지역 소식을 함께 제공하여  
  **정류장을 생활 정보 거점으로 확장**

---

## 주요 사용자

- 버스를 이용하는 일반 시민
- 고령자, 교통 약자
- 관광객 및 외부 방문자
- 스마트폰 사용이 불편한 사용자

---

## 사용 환경 및 특징

- **설치 위치**: 버스 정류장
- **디바이스**: 대형 터치 디스플레이 키오스크
- **사용 방식**
  - 터치 기반 조작
  - 음성 입력 및 음성 안내 지원
- **사용 시간**
  - 짧은 체류 시간
  - 빠른 정보 확인이 핵심

---

## 핵심 기능 요약

### 실시간 버스 정보
- 정류장 기준 도착 예정 버스 표시
- 도착 시간, 혼잡도, 저상버스 여부 제공
- 전체 도착 버스 목록 확인 가능

### 목적지 및 경로 안내
- 목적지 검색 (정류장, 전철역, 주소, 카테고리 등)
- 대중교통 및 도보 경로 제공
- 소요 시간, 환승 정보 안내

### 지도 기반 정보 제공
- 정류장 주변 지도 표시
- 목적지 및 주변 장소 시각화

### 실시간 도로 교통 정보
- 혼잡, 사고, 지연 등 도로 상황 안내

### 지역 정보 제공
- 안양시 소식 및 공지 제공
- 생활 밀착형 정보 제공

---

## 설계 방향

- **한 화면에 하나의 핵심 정보**
- **큰 글자, 명확한 색상 대비**
- **복잡한 입력 최소화**
- **뒤로가기, 닫기 등 명확한 종료 동작 제공**
- **모든 기능은 화면 흐름 기준으로 설계**

---

## 개발 및 구현 기준

- 본 서비스는 화면 중심으로 기능이 구성된다.
- 화면 정의는 `screen-list.md`를 기준으로 한다.
- API 데이터 구조는 `_variableSpec/api-models`의 Python 모델을 기준으로 한다.
- 본 문서는 서비스의 개념과 방향을 설명하는 문서이며,  
  상세 기능 정의는 각 화면 명세 문서를 따른다.
